2.3 Grid-Forming vs Grid-Following

Grid-Following (GFL)
  • Suit la tension/fréquence du réseau
  • Nécessite SCR > 3 (réseau fort)
  • Contrôle en courant (current-source)
  • 90%+ du parc installé actuel
  • Instable si faible inertie
Grid-Forming (GFM)
  • Établit la tension/fréquence
  • Fonctionne en îlotage
  • Virtual Synchronous Machine (VSM)
  • Croissance +40%/an
  • Stable même SCR faible
💬 Perspective Expert: Prof. Frede Blaabjerg (Aalborg, #1 Power Electronics)

"Les Grid-Forming Inverters représentent la révolution nécessaire pour les grids 100% renouvelables. Nos résultats 2025 montrent: fault ride-through <300ms, fréquence ±0.5Hz, RoCoF max 4 Hz/s. Le gap critique reste le déploiement transmission-level et la coordination multi-inverter sans communication haut-débit."

TRL: GFM Distribution=8, GFM Transmission=5-6, Virtual Inertia=7 | Source: CPE-POWERENG 2025 Keynote

2.3.1 Virtual Synchronous Machine

Équation du Swing (VSM)
J × dω/dt = P_ref - P_meas - D × (ω - ω₀)

Où: J = inertie virtuelle, D = amortissement, ω₀ = vitesse nominale
Inertie équivalente: H = J × ω₀² / (2 × S_rated) ≈ 2-6 secondes

3.2 Focus: Stabilité IBR >50%

Défi Critique #1

Au-delà de 50% de pénétration IBR, les microgrids deviennent instables avec les méthodes traditionnelles.

  • Symptômes: ROCOF élevé (>2 Hz/s), oscillations non amorties, échec synchronisation
  • Causes: Absence d'inertie rotative, dynamique rapide IBR, interaction PLL
  • Seuil critique: IBR >50-60% cause instabilité
Solution Émergente: Control Barrier Functions

Les CBF offrent une voie prometteuse pour garantir la sécurité formelle dans les systèmes à forte pénétration IBR. Premières publications CBF+microgrids en 2024-2025 (arXiv:2509.03789, arXiv:2506.03454).

3.3 Focus: Protection Adaptive

Défi Critique #2

Les systèmes de protection traditionnels échouent dans les microgrids car le courant de fault varie d'un facteur 10:1 entre mode grid-connected et islanded.

  • Problèmes: Courant variable, bidirectionnalité, sélectivité compromise
  • Solutions émergentes: Relais adaptatifs, communication-assisted, AI-based detection
💬 Perspective Expert: Prof. Pierluigi Siano (Univ. Salerno, 50k+ citations)

"Les 5 gaps critiques que j'identifie pour les microgrids: (1) Interoperabilité DER-SCADA, (2) Privacy dans Federated Learning, (3) Scalabilité computationnelle pour décisions temps-réel, (4) Robustesse adversariale des AI, (5) Demand Response dynamique proactif. Le gap Academia-Industrie est de 3-5 ans pour les algorithmes IA avancés."

Control Barrier Functions
Fondements théoriques et applications énergétiques

5.1 Fondements Théoriques CBF

Définition Control Barrier Function
Une fonction h: X → ℜ est une CBF pour le système ẋ = f(x) + g(x)u si:

Safe Set: C = {x ∈ X : h(x) ≥ 0}

Condition CBF: supu∈U [Lfh(x) + Lgh(x)u + α(h(x))] ≥ 0

Où α est une fonction de classe K∞

5.1.1 CBF-QP Controller

Quadratic Program pour Safety
minu ||u - uRL||²

s.t. Lfh(x) + Lgh(x)u + α(h(x)) ≥ 0

Résultat: Modification minimale de l'action RL pour garantir sécurité

5.1.2 Propriétés Clés

PropriétéDescriptionImplication
Forward Invariancex(0) ∈ C ⇒ x(t) ∈ C, ∀t≥0Sécurité garantie
Minimal InterventionQP minimise ||u-unom||Performance préservée
Composabilitéh = min(h1, h2, ...)Multi-contraintes
💬 Perspective Expert: Prof. Aaron Ames (Caltech, Pioneer CBF)

"Les 6 problèmes ouverts en CBF: (1) Construction haute dimension (>10 états), (2) Feasibility avec input constraints, (3) Multi-Agent CBF décentralisés, (4) Learning-Based CBF avec garanties, (5) Safe sets time-varying, (6) Compatibilité CBF-CLF. Pour microgrids: les CBF offrent safety sans hardware coûteux - vérification = résoudre un QP. Premières publications CBF+microgrids en 2024-2025."

TRL CBF: Single-agent=7-8, Reduced-order=5-6, Multi-agent=3-4, Power Systems=3-4 | Papers clés: arXiv:2509.03789, arXiv:2506.03454

💬 Perspective Expert: Prof. Stephen Boyd (Stanford, Convex Optimization)

"CBF-QP est un problème convexe élégant. Pour microgrids: SOCP relaxations pour power flow (exact sur réseaux radiaux), ADMM pour optimisation distribuée, Online CVX pour demand response. Le gap: solvers embarqués temps-réel (<10ms). Mon groupe travaille sur code generation automatique pour embedded QP (OSQP, ECOS)."

TRL Solvers: LP=9, QP Embedded=8, SOCP=6-7, SDP=4-5 | Outils: CVXPY, OSQP, ECOS | Investment: $54B batteries 2024

6.2 Modèles SEI, Cycling, Calendar

6.2.1 Modèle SEI Growth

Croissance SEI
dLSEI/dt = kSEI × exp(-Ea / (R × T)) × (1/√t)

Perte capacité: Qloss_SEI = α × LSEI
Facteurs accélérants: T élevée (+10°C = ×2 dégradation), SoC >80%

6.2.2 Modèle Combiné

Dégradation Totale
Qtotal(t) = Q0 × (1 - DSEI(t) - Dcycling(t) - Dcalendar(t))

EOL: Qtotal < 0.80 × Q0
Best Practices
  • SoC Window: 20-80%
  • Température: 15-35°C
  • C-rate: <1C pour longévité
  • Stockage: 50% SoC long terme
Facteurs de Risque
  • +10°C: ×2 vitesse dégradation
  • SoC >80%: SEI accélérée
  • DoD >80%: Stress mécanique
  • Fast charge: Li plating risk
